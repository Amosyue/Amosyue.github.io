2、自己手写一个深拷贝方法

要完成一个深拷贝，我们需要注意以下2点：
1、对于基础类型，我们只需要简单地赋值即可（使用=）。
2、对于引用类型，我们需要创建新的对象，并通过遍历键来赋值对应的值，这个过程中如果遇到 Object 类型还需要再次进行遍历。

3、循环引用

如何解决循环引用呢？我们可以单独开辟一个内存空间存储当前对象和拷贝对象之间的关系，当需要拷贝当前对象的时候就先去内存空间去找,有没有拷贝过当前对象，如果找到了直接返回，如果没找到就继续拷贝。

由于对象的key是不可以是对象的，我们可以采用map这样的数据结构，键值对key-value都可以是对象。

4、复制函数

复制函数，按理说是没有多大意义的，因为js里函数的身份比较特殊，每个函数在声明的时候都处于独立的执行环境（比如闭包），所以你复制一个函数其实是没有多大用处。所以这里我们说的复制函数，其实应该算是复制函数的功能(比如：返回值相同)。

7、总结

如果一个数据很复杂，使用深拷贝会有一定的性能问题，所以在使用时候要根据具体场景选择拷贝的方式。如果是简单的需求用 JSON 反序列化，复杂需求用递归克隆。
